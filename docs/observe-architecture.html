The Problem — Fragmentation Is The Default

Every person who uses AI tools today is multi-provider and multi-agent.

A typical power user runs Claude Code for backend, Cursor for frontend, ChatGPT for research, Codex for async tasks, OpenCode for open-source work — sometimes three of these in the same hour. Each session is a fragment of the user's intent, scattered across tools that don't talk to each other.

⚡ Fragmentation

Session scatter. Each tool has its own session model — Claude Code: JSONL files, OpenCode: SQLite, Codex: JSONL with different schema, ChatGPT: API exports. The user's intent is continuous but the record is fragmented. A decision made in Claude Code at 2am affects work done in OpenCode at 2pm, but no system connects them.

⚙️ Asymmetry

Different tool behaviors. Each tool has its own compaction strategy — Claude truncates context windows silently, OpenCode keeps full history, Codex summarizes. Each records at different granularity — some expose tool calls, some don't. Some track token usage, some hide it.

👁️ Implicit Complexity

What gets lost. The complexity that makes the user's work interesting is exactly what gets lost. If you feed synthesis a single blob per session with no tool calls, no token counts, no agent reasoning traces — the memex will be shallow. Garbage in, garbage out.

🔬 The Research Frontier Is Moving

Recursive Self-Improvement (RSI)

LLMs observing their own execution to train better

Recurrent Loops

Agents that learn from past sessions

Self-Referential Architectures

RLM, ACE, Reflexion — all need operational history as training data

Context engineering research needs real token budgets, cache hit rates, compaction traces. Syke's memory layer works with every harness — but can only be as good as what it sees.

✓ What Observe Solves

A deterministic capture layer that records every turn, every tool call, every tool result, from every harness, with full provenance — at the granularity that makes synthesis and research meaningful. No LLM at capture time. Same input = same events. The adapters absorb the format chaos. The schema stays stable.

🧪 What Sandbox Solves

To model this complex memory system — with its asymmetries, fragmentation patterns, changing tool usage — you need a stable, deterministic environment. Same data, different synthesis models, different prompt conditions. The sandbox replays real data through controlled configurations so you can measure what actually changes.

The data timeline shows when events happened, not when they were captured. The chart reflects the user's actual activity pattern: OpenCode ramping up in Feb, consistent heavy usage through March.

About the Data for Experiments

This is the ONLY dataset. For new sandbox experiments (memory_replay.py), dates can be shifted and events can be sliced by day/source, but the underlying data is this corpus. Each replay experiment creates a fresh DB and copies events day-by-day — the source data is immutable.

02

Observe Layer

The capture layer that absorbs format chaos. Two runtimes handle event capture with different latency and coverage characteristics.

observe-rt Real-Time Runtime

Event-driven capture via hooks, file watchers (FSEvents/inotify), and native streaming protocols. Sub-100ms latency for immediate event ingestion.

observe-sweep Reconciliation Runtime

Periodic full scans (15min default) as safety net. Discovers new sessions, reconciles missed events, handles crash recovery. Ensures completeness.

03

Seven Principles

Constraints that ensure neutral observation across asymmetric sources. Six Observation principles (O1-O6) and seven Design principles (P1-P7) govern system behavior.

Observation Principles

O1

Time Is The Axis

Every event has a timestamp. Monotonic ordering within sessions enables replay and temporal analysis.

O2

Sessions Are Information Units

Never infer session boundaries. Never merge sessions. Each session is a complete, isolated context.

O3

Two Axes

Five event types: turn, tool_call, tool_result, session.start, ingest.error. Complete taxonomy.

O4

Real-Time

Events appear in ledger within seconds of occurrence. Enables live monitoring and reactive synthesis.

O5

Self-Observation

Source='syke' events track synthesis, ingestion, healing. System observes itself for metacognitive awareness.

O6

Healing Is Core

Broken adapter → healed → events resume. Automatic recovery is not optional, it's fundamental.

Design Principles

P1

No Inferred Semantics

No LLM classification, no classifiers, no guessing. Only deterministic, rule-based parsing.

P2

Nullable Over Guessed

NULL is honest. Missing data is better than wrong data. Never fabricate values.

P3

Lossless Provenance

Every event traces to its source. Full audit trail from capture to synthesis.

P4

Raw Preservation

Preserve raw data or apply auditable redaction. No silent transformation or loss.

P5

Append-Only, Ordered

Events are never modified, only appended. Natural ordering by timestamp within source.

P6

Idempotent & Atomic

Same event ingested 5 times = 1 row. external_id deduplication guarantees this.

P7

Failures Are Telemetry

Every parse error, schema drift, and timeout becomes ingest.error event. Observable failures.

Memory Replay Experiment

The experiment runner. Deterministic day-by-day synthesis — replay the corpus, measure what changes.

Source → Replay Pipeline

Copy events day-by-day from production DB to fresh replay DB. Run synthesis after each day. Snapshot memex state.

Tracked Metrics

Characters, sections, pointers, memories, cost per day. Enables ablation studies and synthesis quality analysis.

experiments/memory_replay.py Implementation 
# Memory Replay Experiment Flow

source_db = "syke.db"
replay_db = ":memory:"  # or file for persistence

for day in date_range(start, end):
    # 1. Copy day's events to replay DB
    copy_events(source_db, replay_db, day)
    
    # 2. Run synthesis on accumulated events
    memex = run_synthesis(replay_db)
    
    # 3. Snapshot memex state
    snapshot = {
        "day": day,
        "chars": memex.total_chars,
        "sections": memex.section_count,
        "pointers": memex.pointer_count,
        "memories": memex.memory_count,
        "cost": memex.synthesis_cost
    }
    
    # 4. Save to timeline JSON
    timeline.append(snapshot)

# Ablation conditions: production, no_pointers, neutral
# Outputs: timeline.json + versioned memex snapshots
