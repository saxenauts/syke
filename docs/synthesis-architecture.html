concept

Synthesis is the intelligence layer. It sits on top of the observe layer (the immutable evidence) and builds memory — stories, connections, and a routing map — from raw events. The agent has full bash access to the timeline, 14 memory tools for reading and writing the graph space, and a mandatory completion gate that produces the memex (router) each cycle.

The Three Spaces

Syke operates across three data spaces with different mutability rules.

Space 1: Timeline

Immutable. Read-only.

The events ledger from the observe layer. Append-only. The agent queries it via bash but never modifies it. Self-observation events also land here.

199,138 events · 25+ columns

11 sources · 31 models

5,488 sessions · FTS5 index

Space 2: Graph Space

Mutable. Agent-shaped.

Stories as nodes, connections as edges. The agent creates, updates, supersedes, deactivates memories and links them together. Currently uses typed links with reason field.

717 memories · 382 links

Markdown free-form text

FTS5 index on content

Direction: bare pointers (remove reason field) — discussed, not shipped

Space 3: Router (Memex)

Learned. Dual-readable.

A routing map rewritten each synthesis cycle. Serves humans as a dashboard and agents as a navigation table. Emerged through information pressure.

1 per user · Rewritten each cycle

Control Plane Design

Separating the fixed control plane from the learned layer enables systematic experimentation and measurement.

Current State

The SYNTHESIS_PROMPT is hardcoded in synthesis.py (565 lines). It mixes primitives, instructions, and runtime context in one file.

Target State

Extract into syke_skills/_internal/synthesis-core/SKILL.md

The skill file IS the control plane — primitives and truth, not instructions. Variant overlays will be different wordings of the same skill for ablation testing.

Ablation interpretability: SHA256 hash of rendered skill + variant ID stored on synthesis.start and commit_cycle events. Without this, A/B testing is anecdotal.

Key principle: "Not instructions, primitives. This is the truth. This is what we are measuring."

Why This Separation Matters

Measurability: If the control plane changes every cycle, you can't measure what's working. Fixed primitives let you isolate what the agent is learning.

Ablation: Variant overlays let you test different framings of the same primitives. "map" skill vs "static" skill — same core, different language.

Evolution: GEPA-style prompt optimization requires versioning. Sha256 hash per cycle, stored with results, enables evolutionary search over the control plane.

CURRENT: synthesis.py (565 lines) ├─ Primitives (what to measure) ├─ Instructions (how to behave) └─ Runtime context (per-cycle vars) └─ All mixed together TARGET: Separation of concerns ├─ syke_skills/_internal/synthesis-core/SKILL.md │ └─ FIXED CONTROL PLANE: primitives, truth, measuring stick │ ├─ syke_skills/variants/map/SKILL.md ├─ syke_skills/variants/static/SKILL.md ├─ syke_skills/variants/movement/SKILL.md │ └─ VARIANT OVERLAYS: different wordings, same primitives │ └─ synthesis.py └─ RUNTIME ENVELOPE: injects context, loads skill, runs agent

The Synthesis Agent

A Claude Agent SDK client with full computer access and memory tools. Runs inside _run_synthesis() in synthesis.py (565 lines). The agent receives a system prompt with the current memex, backlog statistics, the database path, and instructions. It queries the timeline, creates/updates memories, and MUST call finalize_memex to complete.

System Prompt Template Variables

{memex_content} — current memex or "[No memex yet]" {backlog_stats} — pending event count + cursor ID {temporal_context} — timezone-aware grounding block {db_path} — full path to SQLite events database {cursor_id} — last processed event ID

Hooks

Stop hook — _enforce_finalize_memex: Scans the transcript for a finalize_memex call. If not found, blocks the agent from stopping and tells it to call finalize_memex now.

PostToolUse hook — _make_self_observe_hook: Records every tool call (tool name + full input) as a synthesis.tool_use event in the events ledger, linked to the cycle's run_id. Added March 18. The feedback loop is confirmed closed.

Observation Gap Analysis

Current coverage: 17% of available signal. Captures tool_name + tool_input only. Missing: tool outputs, tool latency, thinking tokens, cost breakdown, tool failures, ask agent observation. Oracle recommendation: build full trace capture FIRST (Step 1 of Phase 5). Every tool call records: input, output, success/failure, duration. Raw facts on the timeline, derived metrics computed later. Do not invent per-tool cost if SDK doesn't expose it.

Daemon Cycle

The daemon runs every 15 minutes (configurable). It syncs all sources, triggers synthesis if enough new events exist, and distributes the updated memex. Implemented in sync.py (205 lines).

Every 15 minutes (DAEMON_INTERVAL = 900s): ┌─ 1. Health ────── Check all harness connections │ ├─ 2. Heal ──────── Repair broken connections │ ├─ 3. Reconcile ─── Sync all sources via observe adapters │ (each source: detect → discover → ingest → dedup) │ ├─ 4. Synthesize ── If ≥5 new events (SYNC_EVENT_THRESHOLD): │ → synthesize(db, user_id) │ → agent runs, creates memories, writes memex │ └─ 5. Distribute ── Push updated memex to all harnesses: → CLAUDE.md context file → Skills directories → Hermes, Claude Desktop adapters

Cursor & Backlog Management

The synthesis cursor tracks progress through the timeline. It determines what's "new" for each cycle.

synthesis_cursor table: user_id (PK) │ last_event_id │ updated_at Before synthesis: count = events WHERE id > last_event_id ← backlog size newest = MAX(id) FROM events ← will become new cursor Agent receives: "Pending events since last synthesis: {count}" "Cursor (last processed event ID): {cursor_id}" "Query events with: id > '{cursor_id}'" After synthesis: cursor advances to newest event available at cycle start Deterministic — based on what was AVAILABLE, not what was READ Agent decides what to query within the window

Distribution

After synthesis, the memex is distributed to all connected harnesses so every agent has the latest context.

Oracle Risk Assessment (March 19)

  • Silent cursor advancement on incomplete runs — hardest to recover from. No commit_cycle must mean nothing is considered processed.
  • Full trace capture increases volume fast. Default synthesis should exclude source='syke' traces unless explicitly inspecting prior runs.
  • If one-op-per-call becomes chatty, extend memory_write to accept batch arrays. Do not add more tools.
  • If trace payload hurts SQLite ergonomics, move oversized bodies to hashed sidecar blobs. Preserve timeline event as canonical.
