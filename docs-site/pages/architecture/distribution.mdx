# Distribution

How Syke gets your context into every AI tool you use.

## Runtime Distribution Path

<Callout type="info">
Distribution is run from `sync.py` after synthesis. Core entrypoint: `distribute_memex()` in `syke/distribution/context_files.py`.
</Callout>

Flow:

1. `distribute_memex(db, user)` writes `~/.syke/data/{user}/CLAUDE.md` with preamble + memex
2. `ensure_claude_include(user)` ensures `@~/.syke/data/{user}/CLAUDE.md` in `~/.claude/CLAUDE.md`
3. `install_skill()` installs/refreshes `SKILL.md` under:
   - `~/.claude/skills/syke/SKILL.md`
   - `~/.codex/skills/syke/SKILL.md`
   - `~/.cursor/skills/syke/SKILL.md`
   - `~/.windsurf/skills/syke/SKILL.md`
4. Harness refresh (`install_all`) updates detected adapters (Hermes, Claude Desktop)

## Primary Artifacts

```
~/.claude/CLAUDE.md
  └─ @~/.syke/data/{user}/CLAUDE.md

~/.syke/data/{user}/CLAUDE.md          ← current memex + preamble
~/.{tool}/skills/syke/SKILL.md         ← command contract for agents
```

See [Agent Tools Reference](/reference/agent-tools) for the 3 commands available to AI agents.

## Client Targets

<Tabs items={['Claude Code', 'Claude Desktop', 'Codex/Cursor/Windsurf', 'Hermes']}>
<Tabs.Tab>
Automatic include path. New sessions read `~/.claude/CLAUDE.md`, which includes Syke memex.
</Tabs.Tab>
<Tabs.Tab>
Claude Desktop harness adapter updates trusted folders in `claude_desktop_config.json` so local agent mode can access Syke data.
</Tabs.Tab>
<Tabs.Tab>

</Tabs.Tab>
<Tabs.Tab>

- `~/.hermes/skills/memory/DESCRIPTION.md`
- `~/.hermes/skills/memory/syke/SKILL.md`

</Tabs.Tab>
</Tabs>

## Manual Injection (Advanced)

Write context into a specific target directory:

```bash
syke --user you inject --target /path/to/project/.claude --format claude-md
```

This creates a `CLAUDE.md` in the target directory. Claude Code reads it automatically on every session.

## Continuous Sync

The daemon keeps distribution fresh. Every 15 minutes:
1. Re-ingest sources
2. Synthesize memory
3. Run distribution (`distribute_memex`, includes, skills, harness)
4. Persist metrics and status

Identity that drifts with you — no manual re-runs needed.
