# Context Distribution

How Syke gets context into your toolchain after synthesis.

<Callout type="info">
No external MCP deployment required for baseline distribution. Runtime sync writes context files and skill surfaces directly.
</Callout>

## How It Works

```
syke setup
  ↓
1. Setup configures provider + ingests sources
2. Daemon starts (15-minute interval)
3. sync.py runs distribute_memex()
4. Memex written to ~/.syke/data/{user}/CLAUDE.md
5. ensure_claude_include() maintains ~/.claude/CLAUDE.md include
6. Skills installed/updated for Claude, Codex, Cursor, Windsurf
7. Harness adapters refresh Claude Desktop + Hermes integrations
  ↓
Connected tools see current context on next session/run
```

The daemon keeps the memex fresh. Every 15 minutes it collects new signals, runs incremental synthesis if needed, and updates the memex file. Next session sees updated context.

## What Gets Injected

The memex — a synthesized map of who you are:

```markdown
# Memex — dev

## Identity
Full-stack engineer building AI developer tools. Python, TypeScript, React.

## What's Active
- **Syke v0.4.5** [high] (github, claude-code, codex): docs and config hardening
- **Client project** [medium] (gmail, github): API integration due Friday.

## Context
Deep in release mode. Communication style: direct, fast-paced, technical.
Prefers concise answers. Working late nights this week.
```

## Agent Commands

Three commands are available in any Claude Code session:

| Command | What It Does | Cost |
|---------|-------------|------|
| `syke context` | Print the memex (instant, local file read) | Free |
| `syke ask "question"` | Agentic search across memories + timeline | Provider-dependent |
| `syke record "note"` | Push an observation into memory | Free |

See [Agent Tools Reference](/reference/agent-tools) for full documentation.

## Client Compatibility

Syke distribution surfaces by target:

| Client | Support | How |
|--------|---------|-----|
| Claude Code | ✅ Automatic | Reads `~/.claude/CLAUDE.md` with `@includes` |
| Claude Desktop | ✅ Assisted | Trusted folder integration via harness adapter |
| Codex | ✅ Assisted | Skill installed in `~/.codex/skills/syke/SKILL.md` |
| Cursor | ✅ Assisted | Skill installed in `~/.cursor/skills/syke/SKILL.md` |
| Windsurf | ✅ Assisted | Skill installed in `~/.windsurf/skills/syke/SKILL.md` |
| Hermes | ✅ Assisted | Skill installed in `~/.hermes/skills/memory/syke/SKILL.md` |
| Any shell agent | ✅ Direct | Use `syke context`, `syke ask`, `syke record` |

## Keeping Context Fresh

The daemon handles this automatically:

```bash
syke daemon start     # Start background sync
syke daemon status    # Check if running, last sync time
syke sync             # Manual one-time sync
```

Every cycle: collect signals → synthesize if new data → update memex → next session sees it.
