# Getting Started

## Prerequisites

- Python 3.12+

## Install

<Tabs items={['uvx (recommended)', 'pipx', 'From Source']}>
<Tabs.Tab>
```bash copy
uvx syke setup
```
Zero-install path. Runs directly from PyPI via `uvx`.
</Tabs.Tab>
<Tabs.Tab>
```bash copy
pipx install syke
syke setup
```
</Tabs.Tab>
<Tabs.Tab>
```bash copy
git clone https://github.com/saxenauts/syke.git && cd syke
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
python -m syke setup
```
</Tabs.Tab>
</Tabs>

<Callout type="info">
Use providers, not a single API-key path. Syke supports `claude-login`, `codex`, `openrouter`, `zai`, and `kimi`.
</Callout>

## Quick Try: GitHub Only

Fastest path to a working memory state with minimal setup friction:

```bash copy
uvx syke --provider codex setup --yes
```

## Full Pipeline

<Steps>

### Setup

```bash copy
uvx syke setup
```

Setup flow in v0.4.5:
- Step 1: interactive provider picker (`simple-term-menu` arrow-key UI)
- Step 2: source detection + ingestion (`claude-code`, `codex`, `chatgpt`, `github`, optional `gmail`)
- Step 3: daemon install/start (15-minute interval)
- Synthesis is deferred to daemon sync (or manual `syke sync`)

If running non-interactive, set provider with the root-level flag:

```bash copy
syke --provider codex setup --yes
```

## After Setup

<Tabs items={['Provider Already Chosen', 'Change Provider Later', 'Run First Synthesis Now']}>
<Tabs.Tab>
If provider setup succeeded during `syke setup`, all core commands are ready:
- `syke context` вҖ” your synthesized identity profile (instant)
- `syke ask "question"` вҖ” natural language queries over memories + timeline
- `syke record "observation"` вҖ” push observations from any session
</Tabs.Tab>
<Tabs.Tab>
Switch providers at any time:

```bash copy
syke auth status
syke auth use codex
syke auth set openrouter --api-key <key>
syke auth set zai --api-key <key>
syke auth set kimi --api-key <key>
```

Root-level override per invocation:

```bash copy
syke --provider kimi ask "what changed this week?"
```
</Tabs.Tab>
<Tabs.Tab>
The daemon runs every 15 minutes. If you want immediate synthesis:

```bash copy
syke sync
```

Then check current memex:

```bash copy
syke context
```
</Tabs.Tab>
</Tabs>

<Callout type="info">
Context distribution is automatic after sync: Syke writes `~/.syke/data/{user}/CLAUDE.md`, ensures include in `~/.claude/CLAUDE.md`, installs skills for detected clients, and refreshes harness targets.
</Callout>

## Keeping Syke Up to Date

```bash copy
syke self-update          # Upgrade to the latest version from PyPI
```

Syke detects your install method (pipx, pip, uvx, or source) and uses the right upgrade
command. For `uvx`, updates are automatic вҖ” no action needed.

The background daemon checks for new versions once per sync cycle and surfaces an update
available notice in `syke daemon status` and your identity timeline.
