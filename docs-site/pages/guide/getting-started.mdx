# Getting Started

## Prerequisites

- Python 3.12+

## Install

<Tabs items={['uvx (recommended)', 'pipx', 'From Source']}>
<Tabs.Tab>
```bash copy
uvx syke setup --yes
```
Zero-install — runs directly from PyPI via `uvx`.
</Tabs.Tab>
<Tabs.Tab>
```bash copy
pipx install syke
syke setup --yes
```
</Tabs.Tab>
<Tabs.Tab>
```bash copy
git clone https://github.com/saxenauts/syke.git && cd syke
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
cp .env.example .env  # Set ANTHROPIC_API_KEY
python -m syke setup --yes
```
</Tabs.Tab>
</Tabs>

<Callout type="info">
**Synthesis and `ask` need auth — either `ANTHROPIC_API_KEY` or `claude login`.** Claude Code Max/Team/Enterprise subscribers can run `claude login` (works on macOS, Linux, Windows) — no API key needed. [Get an API key here](https://console.anthropic.com/settings/keys) if you prefer pay-per-use. Without either, data collection and the daemon still work — synthesis and `ask` are skipped until auth is configured.
</Callout>

<Callout type="info">
**AI Agent Setup.** Paste `https://github.com/saxenauts/syke` into any AI agent and say "install this." Two commands, everything works.
</Callout>

## Full Pipeline

<Steps>

### Setup

```bash copy
uvx syke setup --yes
```

This auto-detects your username and ingests:
- Claude Code sessions from `~/.claude/`
- ChatGPT export ZIPs in `~/Downloads/`
- GitHub via `gh` CLI or `GITHUB_TOKEN`
- Gmail via OAuth (if configured)

Then runs memory synthesis and outputs your profile.

### Verify

```bash copy
uvx syke status
```

## After Setup

<Tabs items={['With API Key', 'With claude login', 'No Auth']}>
<Tabs.Tab>
If `ANTHROPIC_API_KEY` is set, setup collects data **and** builds your identity profile automatically. All agent commands work — including `syke ask` and `syke context`.
Key commands available:
- `syke context` — your synthesized identity profile (instant)
- `syke ask "question"` — natural language queries about you (~$0.10/call)
- `syke record "observation"` — push observations from any session
</Tabs.Tab>
<Tabs.Tab>
Claude Code Max/Team/Enterprise subscribers can use `claude login` instead of an API key. Works on macOS, Linux, and Windows.

```bash copy
claude login
uvx syke setup --yes
```

Synthesis and `ask` use your subscription — no API key needed. All 3 agent commands work immediately.
</Tabs.Tab>
<Tabs.Tab>
Without any auth, setup collects all your data and starts the daemon. `syke context` and `syke record` work immediately.

To enable synthesis and `ask`, add auth:
```bash copy
# Option A: API key
export ANTHROPIC_API_KEY=sk-ant-...

# Option B: Claude Code Max/Team/Enterprise
claude login
```

Then re-run sync:
```bash copy
syke sync              # Run in a standalone terminal, not inside Claude Code
```
</Tabs.Tab>
</Tabs>

<Callout type="info">
**That's it.** Context is distributed via CLAUDE.md file includes — every new Claude Code session automatically reads your memex on startup. No restart needed after the first `syke setup`.
</Callout>

## Keeping Syke Up to Date

```bash copy
syke self-update          # Upgrade to the latest version from PyPI
```

Syke detects your install method (pipx, pip, uvx, or source) and uses the right upgrade
command. For `uvx`, updates are automatic — no action needed.

The background daemon checks for new versions once per sync cycle and surfaces an update
available notice in `syke daemon status` and your identity timeline.
