# Getting Started

## Prerequisites

- Python 3.12+

## Install

<Tabs items={['uvx (recommended)', 'pipx', 'From Source']}>
<Tabs.Tab>
```bash copy
uvx syke setup --yes
```
Zero-install — runs directly from PyPI. This is the pattern every official MCP server uses.
</Tabs.Tab>
<Tabs.Tab>
```bash copy
pipx install syke
syke setup --yes
```
</Tabs.Tab>
<Tabs.Tab>
```bash copy
git clone https://github.com/saxenauts/syke.git && cd syke
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
cp .env.example .env  # Set ANTHROPIC_API_KEY
python -m syke setup --yes
```
</Tabs.Tab>
</Tabs>

<Callout type="info">
**Perception and `ask()` need auth — either `ANTHROPIC_API_KEY` or `claude login`.** Claude Code Max/Team/Enterprise subscribers can run `claude login` (works on macOS, Linux, Windows) — no API key needed. [Get an API key here](https://console.anthropic.com/settings/keys) if you prefer pay-per-use. Without either, data collection, MCP timeline tools, and the daemon still work — perception and `ask()` are skipped until auth is configured.
</Callout>

<Callout type="info">
**AI Agent Setup.** Paste `https://github.com/saxenauts/syke` into any AI agent and say "install this." Two commands, everything works.
</Callout>

## Full Pipeline

<Steps>

### Setup

```bash copy
uvx syke setup --yes
```

This auto-detects your username and ingests:
- Claude Code sessions from `~/.claude/`
- ChatGPT export ZIPs in `~/Downloads/`
- GitHub via `gh` CLI or `GITHUB_TOKEN`
- Gmail via OAuth (if configured)

Then runs agentic perception and outputs your profile.

### Verify

```bash copy
uvx syke status
```

Expected output:

```
╭──────────────── Syke Status: alice ──────────────╮
│ Sources: claude-code, chatgpt, github             │
│ Events:  150 total                                │
│ Profile: Generated 2026-02-15 14:30               │
│ MCP:     Configured (stdio)                       │
╰───────────────────────────────────────────────────╯
```

</Steps>

## After Setup

<Tabs items={['With API Key', 'With claude login', 'No Auth']}>
<Tabs.Tab>
If `ANTHROPIC_API_KEY` is set, setup collects data **and** builds your identity profile automatically. After restarting Claude Code, all MCP tools work — including `ask()` and `get_profile()`.

Key MCP tools available:
- `ask(question)` — natural language queries about you (~$0.02/call)
- `get_profile()` — your identity profile
- `query_timeline()` / `search_events()` — browse your timeline (no API cost)
- `push_event()` — record observations from any session
</Tabs.Tab>
<Tabs.Tab>
Claude Code Max/Team/Enterprise subscribers can use `claude login` instead of an API key. Works on macOS, Linux, and Windows.

```bash copy
claude login
uvx syke setup --yes
```

Perception and `ask()` use your subscription — no API key needed. All 8 MCP tools work after restarting Claude Code.
</Tabs.Tab>
<Tabs.Tab>
Without any auth, setup collects all your data and configures the MCP server. The 7 timeline tools (`get_profile`, `query_timeline`, `search_events`, `get_event`, `get_manifest`, `push_event`, `push_events`) work immediately after restart.

To enable perception and `ask()`, add auth:
```bash copy
# Option A: API key
export ANTHROPIC_API_KEY=sk-ant-...

# Option B: Claude Code Max/Team/Enterprise
claude login
```

Then re-run sync and restart Claude Code:
```bash copy
syke sync --rebuild    # Run in a standalone terminal, not inside Claude Code
```
</Tabs.Tab>
</Tabs>

<Callout type="warning">
**Restart Claude Code** after setup to activate the MCP server. The tools won't appear until you start a new session.
</Callout>

## Keeping Syke Up to Date

```bash copy
syke self-update          # Upgrade to the latest version from PyPI
```

Syke detects your install method (pipx, pip, uvx, or source) and uses the right upgrade
command. For `uvx`, updates are automatic — no action needed.

The background daemon checks for new versions once per sync cycle and surfaces an update
available notice in `syke daemon-status` and your identity timeline.
