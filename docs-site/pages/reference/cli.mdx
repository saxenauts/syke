# CLI Reference

Command shape:

`syke [global-options] <command> [command-options]`

Global options:

- `--user`, `-u` user id (default from config/env)
- `--verbose`, `-v` verbose logging
- `--provider`, `-p` provider override for this invocation
- `--version`

<Callout type="warning">
`--provider` is root-level. Correct: `syke --provider codex setup`. Incorrect: `syke setup --provider codex`.
</Callout>

## Common Workflows

<Steps>

### First-time setup

```bash
uvx syke setup
```

Or with persistent install: `pipx install syke && syke setup`

### Non-interactive setup with explicit provider

```bash
syke --provider codex setup --yes
```

### Add a new source + sync

```bash
syke --user you ingest github --username your-username
syke --user you sync
```

### Check status

```bash
syke --user you status
syke doctor
```

</Steps>

## setup

```bash
syke setup [--yes/-y] [--skip-daemon]
```

Flow:

1. Provider selection (interactive picker unless root `--provider` is supplied)
2. Source detection + ingestion (`claude-code`, `codex`, `chatgpt`, `github`, optional `gmail`)
3. Daemon start prompt (or automatic with `--yes`)
4. First synthesis runs on daemon tick (or `syke sync`)

## status

```bash
syke status
```

Shows source event counts, recent ingestion runs, and memex status.

## sync

```bash
syke sync
```

Runs source sync, synthesis, and distribution refresh.

## ask

```bash
syke ask "what changed this week?"
```

Agentic memory query over memex + evidence.

## context

```bash
syke context [--format markdown|json]
```

Prints current memex to stdout.

## record

```bash
syke record "text"
syke record --tag work --source manual "text"
echo '{"text":"note","source":"agent"}' | syke record --json
cat events.jsonl | syke record --jsonl
```

Options:

- `--tag`, `-t` repeatable tags
- `--source`, `-s` source label (default `manual`)
- `--title` custom title
- `--json` parse one JSON event
- `--jsonl` parse newline-delimited JSON events from stdin

## auth

```bash
syke auth status
syke auth set <provider> --api-key <key>
syke auth use <provider>
syke auth unset <provider>
```

Supported providers: `claude-login`, `codex`, `openrouter`, `zai`, `kimi`.

## config

```bash
syke config show
syke config show --raw
syke config init [--force]
syke config path
```

Manages optional `~/.syke/config.toml`.

## doctor

```bash
syke doctor [--network]
```

Checks provider resolution, daemon status, database health, and harness adapter state.

## self-update

```bash
syke self-update [--yes]
```

Updates from PyPI using install-method-aware behavior.

## daemon

```bash
syke daemon install
syke daemon start [--interval 900]
syke daemon status
syke daemon stop
syke daemon uninstall
```

Also available:

```bash
syke daemon logs [-n 50] [-f] [--errors]
```

## ingest (advanced)

```bash
syke ingest claude-code [--yes]
syke ingest codex [--yes]
syke ingest chatgpt --file <zip> [--yes]
syke ingest github --username <name>
syke ingest gmail [--yes] [--account <email>] [--max-results N] [--days N] [--query <q>]
syke ingest all [--yes]
```

`ingest` is an advanced/manual path; most users should use `syke setup` and `syke sync`.

## Provider Override Examples

```bash
syke --provider codex setup --yes
syke --provider openrouter sync
syke --provider kimi ask "summarize current priorities"
```
