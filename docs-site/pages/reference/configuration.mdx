# Configuration

Syke configuration is optional and layered:

hardcoded defaults -> `~/.syke/config.toml` -> environment variables.

## Config File

Path: `~/.syke/config.toml`

Generate a template:

```bash
syke config init
```

Inspect effective runtime configuration:

```bash
syke config show
syke config show --raw
```

## Full `config.toml` Schema

```toml
# identity
user = "alice"
timezone = "auto"
provider = "codex"

[models]
synthesis = "sonnet"
ask = ""
rebuild = "opus"

[sources]
claude-code = true
codex = true
chatgpt = true
gmail = false

[sources.github]
enabled = true
username = "alice"

[synthesis]
budget = 0.50
max_turns = 10
threshold = 5
thinking = 2000
first_run_budget = 2.00
first_run_max_turns = 25

[daemon]
interval = 900

[ask]
budget = 1.00
max_turns = 8
timeout = 120

[rebuild]
budget = 3.00
max_turns = 20
thinking = 30000

[distribution]
claude-code = true
claude-desktop = true
hermes = true

[privacy]
redact_credentials = true
skip_private_messages = true

[paths]
data_dir = "~/.syke/data"
auth = "~/.syke/auth.json"

[paths.sources]
claude_code = "~/.claude"
codex = "~/.codex"
chatgpt_export = "~/Downloads"

[paths.distribution]
claude_md = "~/.claude/CLAUDE.md"
skills_dirs = [
  "~/.claude/skills",
  "~/.codex/skills",
  "~/.cursor/skills",
  "~/.windsurf/skills",
]
hermes_home = "~/.hermes"
```

## Environment Variables

Set in `.env` at the project root:

<Callout type="info">
Auth is provider-based in v0.4.5. `claude-login`, `codex`, `openrouter`, `zai`, and `kimi` are supported.
</Callout>

| Variable | Required | Description |
|----------|----------|-------------|
| `SYKE_PROVIDER` | No | Provider override for current process |
| `SYKE_OPENROUTER_API_KEY` | No | OpenRouter token |
| `SYKE_ZAI_API_KEY` | No | z.ai token |
| `SYKE_KIMI_API_KEY` | No | Kimi token |
| `GITHUB_TOKEN` | No | GitHub PAT for private repos |
| `GMAIL_ACCOUNT` | No | Gmail address for `gog` CLI backend (e.g., `you@gmail.com`) |
| `GMAIL_CREDENTIALS_PATH` | No | Path to Gmail OAuth credentials (Python backend) |
| `GMAIL_TOKEN_PATH` | No | Path to Gmail OAuth token (Python backend) |
| `SYKE_USER` | No | Default user ID (avoids `--user` flag) |
| `SYKE_DATA_DIR` | No | Data directory (default: `~/.syke/data`) |

Common tuning env vars:

| Variable | Default | Description |
|----------|---------|-------------|
| `SYKE_SYNC_MODEL` | `sonnet` | Synthesis model alias |
| `SYKE_ASK_MODEL` | provider default | Ask model override |
| `SYKE_REBUILD_MODEL` | `opus` | Rebuild model alias |
| `SYKE_SYNC_BUDGET` | `0.50` | Synthesis budget (USD) |
| `SYKE_ASK_BUDGET` | `1.00` | Ask budget (USD) |
| `SYKE_REBUILD_BUDGET` | `3.00` | Rebuild budget (USD) |
| `SYKE_DAEMON_INTERVAL` | `900` | Daemon interval seconds |
| `SYKE_SYNC_THRESHOLD` | `5` | Min new events before synthesis |

Provider precedence: root CLI `--provider` -> `SYKE_PROVIDER` -> `~/.syke/auth.json` active provider -> auto-detected `claude-login`.

## Gmail Configuration

The Gmail adapter automatically picks the best available backend:

<Tabs items={['gog CLI (recommended)', 'Python OAuth']}>
<Tabs.Tab>
Fast, handles keychain natively.

```bash
brew install gog
gog auth add --account you@gmail.com
```

Set `GMAIL_ACCOUNT=you@gmail.com` in `.env`. The adapter calls `gog` directly.
</Tabs.Tab>
<Tabs.Tab>
Self-contained, no extra CLI needed.

```bash
pip install google-auth-oauthlib google-api-python-client
```

Download OAuth credentials from [Google Cloud Console](https://console.cloud.google.com/apis/credentials) and save to `~/.config/syke/gmail_credentials.json`. On first run, a browser window opens for consent. The token is cached at `~/.config/syke/gmail_token.json`.

## File Locations

| What | Where |
|------|-------|
| Configuration file | `~/.syke/config.toml` |
| Auth store | `~/.syke/auth.json` |
| User data | `~/.syke/data/{user_id}/` |
| SQLite database | `~/.syke/data/{user_id}/syke.db` |
| CLAUDE.md (memex) | `~/.syke/data/{user_id}/CLAUDE.md` |
| Metrics log | `~/.syke/data/{user_id}/metrics.jsonl` |
| Application log | `~/.syke/data/{user_id}/syke.log` |

## Auth Store Details

`~/.syke/auth.json` is plaintext JSON with:

- schema version
- active provider id
- provider tokens

Writes are atomic (temp file + rename), permissioned to `0600`, and guarded by `flock` for concurrent CLI/daemon safety.

## Model Defaults

Defaults from typed dataclasses in `syke/config_file.py`:

- synthesis model `sonnet`, budget `$0.50`, max turns `10`
- ask budget `$1.00`, max turns `8`, timeout `120s`
- rebuild model `opus`, budget `$3.00`, max turns `20`
