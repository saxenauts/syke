# Agent Tools Reference

Syke exposes 3 runtime CLI commands for agents and humans, backed by a 15-tool internal memory toolset used by synthesis and ask flows.

<Callout type="info">
**Pattern**: `syke context` first → `syke ask` if needed → `syke record` when something happens.

`context` and `record` are local. `ask` needs a configured provider (`claude-login`, `codex`, `openrouter`, `zai`, or `kimi`).
</Callout>

## syke context

**Start here.** Returns the memex snapshot. Instant local read.

Not a raw DB dump. This is synthesized memory, refreshed on sync.

```bash
syke context                    # Print memex to stdout
syke context --format json      # JSON format
```

**When to use:**
```
"What is the user working on?"           → syke context (it's right there)
"How do they prefer error messages?"     → syke context (communication style)
"What did they think about X 3 weeks ago?" → syke ask (needs timeline exploration)
```

---

## syke ask

Ask any question about the user in natural language. Syke's internal agent reads the memex, then explores the memory layer and raw timeline to find the answer.

More thorough than direct context reads, but slower because it runs an agentic search loop.

**Requires:** a configured provider.

```bash
syke ask "What did they work on last week?"
syke ask "What do they think about RAG architectures?"
syke ask "Find the decision about JWT vs session tokens"
```

Use `ask` for questions requiring evidence navigation or temporal reconstruction.

---

## record

Push a meaningful observation back into the user's timeline. Natural language, no structure needed.

The daemon already captures sessions, commits, and conversations automatically. Use `record` for signals the daemon can't see: decisions made in conversation, preferences expressed, frustrations, insights.

```bash
syke record "User decided to use PostgreSQL over MongoDB"
syke record "Finished the auth refactor and merged PR #42"
syke record "Prefers dark mode, mentioned it twice today"
```

**When to call:** Decisions, task completions, strong preferences, new collaborations, project shifts.
**When NOT to call:** Every message (too noisy), trivial observations, info already in the live context.

## Internal Synthesis/Ask Tooling

Syke's internal memory tool registry (15 tools):

- Read/search tools: `search_memories`, `search_evidence`, `follow_links`, `get_recent_memories`, `get_memex`, `browse_timeline`, `cross_reference`, `get_memory`, `list_active_memories`, `get_memory_history`
- Mutation tools: `create_memory`, `create_link`, `update_memory`, `supersede_memory`, `deactivate_memory`

Behavior notes:

- Search uses SQLite FTS/BM25 ranking, not vector retrieval
- Event and memory previews are truncated for tool responses
- `ask` uses read/navigation paths to answer; synthesis uses mutation paths to evolve memory
- Supersession keeps history chains instead of destructive overwrites

## How Distribution Works

Syke distributes context via file includes — no MCP server needed:

```
distribute_memex() → writes ~/.syke/data/{user}/CLAUDE.md (memex + preamble)
ensure_claude_include() → maintains @include in ~/.claude/CLAUDE.md
install_skill() → updates skill files in Claude/Codex/Cursor/Windsurf skill dirs
harness.install_all() → refreshes Hermes/Claude Desktop adapters where detected
```

Claude Code gets automatic include-based preload; other clients consume via installed skill surfaces and CLI commands.
